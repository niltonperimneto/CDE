# Altre attività generali


La finestra della Gestione di applicazioni ha un funzionamento molto simile a quella della Gestione di file. Potrà perciò essere utile consultare anche le seguenti sezioni dell'aiuto sulla Gestione di file.

Selezionare un unico file o un'unica cartella

Trascinare e rilasciare un file o una cartella

Spostare un file o una cartella

Copiare un file o una cartella

Cambiare le impostazioni di visualizzazione

Per ulteriori informazioni sull'utilizzo della Gestione di file, vedere:Attività della Gestione di file.

# Aggiornare la Gestione di applicazioni
Gestione di applicazioni:aggiornamentoaggiornamento delle applicazioniapplicazioni:aggiornamento

La Gestione di applicazioni viene creata la prima volta che si effettua il login e ricreata ad ogni login successivo. Durante questa procedura, le applicazioni che si trovano in determinate posizioni del sistema e della rete vengono inserite nella Gestione di applicazioni.

Se si desidera aggiornare la Gestione di applicazioni dopo avere aggiunto un'applicazione, procedere come segue:

Eseguire il logout e quindi un nuovo login.

Oppure, aprire il gruppo di applicazioni Strumenti_desktop e
fare doppio clic su Ricaricare applicazioni.
# Creare un gruppo di applicazioni personale
gruppi di applicazioni:personaliGestione di applicazioni:modifica

Si definisce personale un gruppo di applicazioni che appartiene esclusivamente all'utente che lo crea. L'utente è l'unico a possedere le autorizzazioni per modificarlo.

Creare una directory:`DirectoryIniziale`/.dt/appmanager/`nome_gruppo`

dovenome_gruppoè il nome del gruppo di applicazioni che dovrà comparire nella Gestione di applicazioni.

Aprire il gruppo di applicazioni Strumenti_desktop e fare doppio clic su
Ricaricare applicazioni.

A questo punto il nuovo gruppo di applicazioni sarà visibile nel livello
più elevato della Gestione di applicazioni.

Il gruppo di applicazioni personale potrà essere trattato come qualsiasi altra directory alla quale si ha accesso in scrittura. È possibile utilizzarla come cartella di destinazione per lo spostamento, la copia o la cancellazione di file, oppure per la creazione di directory secondarie.

# Aggiungere un'applicazione a un gruppo di applicazioni &newline; personale
Gestione di applicazioni:modificaapplicazioni:aggiunta

La procedura seguente:

Crea un'azione per l'applicazione.

Le azioni sono "programmi" del desktop, paragonabili alle macro, che consentono di automatizzare le operazioni eseguite più frequentemente dalla riga comandi. Il desktop utilizza le azioni per associare le righe comandi alle icone.

Crea un'icona per l'azione che rappresenti l'applicazione nel gruppo.

Se necessario, creare un gruppo di applicazioni personale. Vedere.

Determinare il comando da utilizzare per avviare l'applicazione da una finestra di emulatore di terminale.

Creare azione

Creare l'azione appropriata usando l'applicazione Creare azione.
VedereCreare un'azione con Creare azione.

Creare azione richiederà di inserire un nome per l'azione. Il
nome specificato sarà utilizzato come etichetta per l'azione.

A questo punto, Creare azione creerà quindi un'icona per l'azione nella cartella iniziale
dell'utente. Questa icona è un file eseguibile che ha lo stesso
nome dell'azione.

Copiare o spostare l'icona della nuova azione dalla cartella iniziale
al gruppo di applicazioni personale.
# Registrare un'applicazione nella Gestione di applicazioni
applicazioni:registrazioneregistrazione di applicazioniGestione di applicazioni:modifica

Quando un'applicazione viene registrata nella Gestione di applicazioni, essa viene inclusa automaticamente in un determinato gruppo. Tale gruppo sarà disponibile a tutti gli utenti del sistema.

La registrazione delle applicazioni viene svolta di norma dall'amministratore del sistema. Per le relative istruzioni, vedere il manualeManuale del Common Desktop Environment per utenti esperti e amministratori di sistema.
